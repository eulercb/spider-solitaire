# Baize — Spider Solitaire

A refined night-time card parlor in your pocket. Spider Solitaire as an
installable, fully-offline PWA for Android/Chrome, hosted on GitHub Pages.

**Play:** `https://<user>.github.io/spider-solitaire/` (deploys from `main`
via GitHub Actions — enable Pages → Source: *GitHub Actions* once).

## What's inside

- **1 / 2 / 4-suit Spider**, exact classic rules: loose placement (any suit,
  one rank higher), strict group pickup (same-suit descending runs only),
  ten-card stock deals blocked while a column is empty, automatic K→A
  sequence removal, eight sequences to win.
- **Two interaction modes at once** — drag (cards ride above the finger,
  legal targets glow, snap back on miss) and tap-to-move (best destination:
  same-suit continuation, then occupied piles, then empty columns).
- **Unlimited undo/redo**, hint, restart-this-deal (seeded), auto-finish
  offer when the endgame is provably won.
- **GSAP choreography** for deal, move, flip, sequence-clear, and invalid
  shake — interruptible, engine-state-first, `prefers-reduced-motion` aware.
- **PixiJS WebGL win cascade** (lazy-loaded, `pointer-events: none`, WebGL
  feature-detected) with a CSS fallback celebration.
- **Baize art direction**: deep felt + woven grain + vignette, brass
  accents, Fraunces/Inter (self-hosted), Midnight theme, three card backs,
  four-color and high-contrast decks, large-index option.
- **Autosave + resume**, per-difficulty stats with streaks, versioned
  storage that forgives corrupt data.
- Haptics (on) and synthesized WebAudio sounds (off) — no audio assets.

## Development

```bash
npm install
npm run dev        # local dev server
npm test           # engine rules test suite (Vitest)
npm run build      # type-check + production build to dist/
npm run smoke      # build first; Playwright end-to-end smoke + screenshots
npm run icons      # regenerate PNG icons from the SVG art
```

The rules engine (`src/engine/`) is pure TypeScript — no DOM, no side
effects, fully covered by tests. The DOM view reconciles to engine state;
animations are downstream of committed state, so interrupting them can
never desync the game. See `DECISIONS.md` for deliberate deviations.

## Deployment

`.github/workflows/deploy.yml` tests, builds, and publishes `dist/` to
GitHub Pages on every push to `main`. Vite's `base` is `/spider-solitaire/`;
the manifest uses relative `start_url`/`scope`, and the service worker
(`vite-plugin-pwa`, `autoUpdate`) precaches the entire shell — fonts, card
art, and the Pixi chunk — for true offline play.
